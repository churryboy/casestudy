# 케이스스터디 문서 규칙 (Case Study Document Rules)

> **용도:** 보고서(1)가 작성된 뒤, 이 규칙을 적용해 **케이스스터디에 최적화된 문서(2)**로 재정리할 때 사용한다.  
> **참고 사례:** `case-study-02-local-government.html`, `case-study-03-japan-fashion.html`

---

## 1. 문서의 목적과 톤

- **독자:** 외부 방문자(잠재 고객·파트너), Proby 리서치 가치를 이해하려는 사람
- **목표:** "무엇을 했고, 어떤 인사이트·제언을 냈는지"를 **빠르게 스캔**할 수 있게 정리
- **톤:** 전문적이되 과하지 않게. **핵심 문장 + 인용(VOC) + 번호·제목**으로 읽기 부담을 줄인다.
- **금지:** 원본 보고서의 장황한 서론·방법론 세부·표/부록 나열을 그대로 옮기지 않는다.

## 3. 섹션별 상세 규칙

### 3.1 히어로 (상단)

- **카테고리 배지:** 한 줄, 대문자·짧은 라벨. 예: `지자체 · 광고 주체 리서치`, `패션 · 이커머스 · 일본 시장`
- **제목 (h1):** 프로젝트명. "~ 조사", "~ 방향성", "~ 인사이트" 등 **결과·테마가 드러나는** 문구.
- **한 줄 요약:** 1~2문장. "무엇을 했고 → 어떤 인사이트/제언을 냈는지" 요약. 독자가 5초 안에 이해할 수 있는 수준.

**제목 규칙 (외부 공개용):**
- **원칙:** 외부인·일반인이 보더라도 이해할 수 있는 표현. **실무자 내부 문서 같은 제목은 사용하지 않는다.**
- **지양:** "○○ 조사 1~7 통합 분석", "○○ POC 2차 보고서"처럼 **과제 번호·차수·내부 프로젝트명**이 들어간 제목.
- **선호:** "MZ 세대 금융 라이프스타일 및 은행 이용 인사이트", "지자체 주관 행사 참여 경험 조사", "일본 유저가 선호하는 패션 앱 UI 방향성"처럼 **무엇을 다룬 조사/무슨 결과인지**가 드러나는 제목.
- 한 줄 요약도 **MECE, POC, 차수** 등 내부 용어 없이, "무엇을 파악했고 → 어떤 제언을 냈는지"만 담는다.

**재정리 시 체크:**
- [ ] 배지는 업종·도메인·리서치 유형이 드러나게
- [ ] 제목은 구체적이되 **내부 문서형(과제명·차수·통합분석 등)이 아닌** 결과/테마 중심
- [ ] 한 줄 요약에 "도출", "파악", "제언" 등 결과 동사 포함

### 3.2 개요 (overview)

**구성:**
1. **도입 단락 (1개):** 프로젝트 배경 + 목표 + "무엇을 위한 조사인지" 2~4문장.
2. **요약 카드 (2~4개):** 그리드 형태. 각 카드마다:
   - **아이콘/라벨** (선택): 예) 조사 목적, 참여자 조건, 참석 행사, 산출물
   - **제목 (h4):** 예) "조사 목적", "참여자 조건", "다룬 주제와 결과물"
   - **본문:** 1~3문장. 숫자(참여자 수)는 명시하되, **외부인이 이해할 수 있는 말**로만 서술.

**개요 표현 규칙 (외부인 이해 가능):**
- **원칙:** 내부자만 아는 용어·약어·과제 코드를 쓰지 않고, **아무나 읽어도 이해되는 문장**으로 쓴다.
- **지양:** "○○ 조사 1~7", "리서치랩 POC", "[Desired insight]", "MECE 7축", "LOI", "인터뷰 1~7" 등 **내부 프로젝트명·분석 프레임명·참여자 번호**를 그대로 노출. "정성 인사이트 도출 및 캠페인 제언 정리"처럼 **실무 문서 톤**의 나열.
- **선호:** "7명의 20~40대", "금융 생활 전반을 서로 겹치지 않는 7개 영역으로 나누어", "참여자들이 금융에 얼마나 관심 있는지, … 깊이 있게 듣고 제안하는 데 목적"처럼 **누가, 무엇을, 왜**가 드러나는 일상어.
- 카드 제목도 "MECE 7축 및 산출물" → "다룬 주제와 결과물", "참여자 조건" 본문에서 "(인터뷰 1~7)" 생략·"7명, 20~40대, 직장·자영업 포함" 등으로만 서술.
- 방법론·출처 문구에서도 "금융행태 조사 1~7 가이드라인", "LOI 24분" 대신 "금융 생활과 은행 이용에 대한 질문을 바탕으로", "평균 약 24분" 등으로 치환.

**재정리 시 체크:**
- [ ] 도입 단락에 "목표", "대상", "결과"가 들어가되 **내부 용어 없이**
- [ ] 카드 제목·본문 모두 **외부인이 한 번에 이해 가능한 표현**인지 확인
- [ ] 참여자 수·연령대·특성은 명시, 인터뷰 번호(1~N)는 필요 시 본문(VOC)에만
- [ ] 카드 개수는 2~4개로 제한(과하지 않게)

---

### 3.3 연구 방법론 (methodology)

**구성:**
- **h2:** "연구 방법론"
- **방법 블록 1개(또는 소수):**
  - **제목 (h4):** 예) "AI 인터뷰 (Proby)", "AI 심층 인터뷰 (Proby)"
  - **설명 단락:** 어떻게 진행했는지(대상, 절차, 자극물 유무 등) 2~4문장.
  - **보조 설명 (선택):** 인용 스타일(italic, 왼쪽 border)로 "참여자 반응 요약" 또는 "진행 방식 보충(예: 자극물 사전 전달)" 1문장.

**재정리 시 체크:**
- [ ] 방법론은 "한눈에 이해" 수준으로 압축. 세부 설계·표는 생략 가능
- [ ] Proby/AI 인터뷰 사용 시 "1:1 대화형", "맥락에 맞는 후속 질문" 등 차별점 한 줄 포함
- [ ] 참여자 피드백(예: "압박감 없이 편하다")이 있으면 보조 설명으로

---

### 3.4 주요 발견사항 (findings)

**구성:**
1. **퀵 인용 블록 (선택):** 상단에 검은 배경·흰 글씨 인용 1개. "— 인터뷰 N" 또는 "— 일본 참여자 발화" 형태.
2. **발견 블록들 (필수):** 각 블록마다:
   - **제목 (h4):** 번호 + 핵심 주제. 예) "1. 행사 인지 채널: 채널별 언급 빈도와 복합 경로"
   - **본문:** 2~5문장. **굵은 글씨**로 키워드(예: 가성비·리뷰·이색 체험) 강조.
   - **인용(VOC) 0~2개:** 참여자 발화를 italic로, 필요 시 "(인터뷰 N)" 표기.
   - **미디어 (선택):** 영상/이미지가 있으면 제목(h5) + 파일 경로 명시.

**재정리 시 체크:**
- [ ] 발견사항은 "번호 + 한 줄 제목"으로 스캔 가능하게
- [ ] 본문은 인사이트 중심. 원문의 긴 서론·반복은 제거
- [ ] **VOC(참여자 발화)는 반드시 1개 이상** 넣어 신뢰도·생생함 확보
- [ ] 키워드는 **strong(굵게)** 처리해 요약 검색에 유리하게
- [ ] 하이라이트 영상 등이 있으면 "제목 + 경로"만 규칙 문서에 기록(HTML 변환 시 삽입)

---

### 3.5 도메인별 추가 섹션 (선택)

- 프로젝트 성격에 따라 **findings와 impact 사이**에 추가.
- 예: "일본 고유 정서", "재방문 의사", "채널별 인지 경로" 등.
- **규칙:** 제목(h2) + 짧은 설명 단락 + 카드/리스트 형태로 정리. 가능하면 "한 줄 요약" 박스로 마무리.

---

### 3.6 제언 및 시사점 (impact)

**구성:**
- **h2:** "향후 캠페인을 위한 제언" 또는 "Product Designer를 위한 제언" 등 **수혜자 기준** 제목.
- **제언 블록:** 대제목(h3) + 소제목(h4) + 1~2문장. 예) "채널·메시지·정보 제공" → "채널: 첫 인지와 검증 분리 설계"
- **Must-have / 체크리스트:** 제안 사항을 불릿·짧은 문장으로 나열해도 됨.
- **한 문장 요약 (권장):** "한 줄 요약:", "한 문장 요약:" 등으로 클라이언트가 외우기 쉬운 메시지 1개.
- **출처 문구 (필수):** "본 케이스는 ~ 내용을 종합하여 작성되었습니다." (인터뷰 N명, 보고서명 등)

**재정리 시 체크:**
- [ ] 제언은 "누가 쓸 수 있는지" 명확히(캠페인 기획자, PD 등)
- [ ] 제언은 구체적 행동 가능한 수준(채널 설계, 메시지 톤, 정보 제공 방식 등)
- [ ] 출처 문구로 원본 보고서/인터뷰 범위 명시

## 4. 문체·표현 규칙

| 항목 | 규칙 |
|------|------|
| 문장 길이 | 한 문장은 가능하면 2줄 이내. 끊어 읽기 쉬우면 OK |
| 숫자 | 참여자 수, 인터뷰 번호(1~7 등)는 반드시 명시 |
| 인용 | 참여자 발화는 따옴표 + 이탤릭, 끝에 "— 인터뷰 N" 또는 "— 일본 참여자 발화" |
| 강조 | 핵심 키워드는 **굵게** |
| 금지 | "~하였다", "~해 보았다" 등 과한 격식. "~했습니다", "~도출했습니다" 등 자연스러운 경어 |

## 5. 재정리 워크플로우 체크리스트

1. [ ] 원본 보고서에서 **목적·대상·방법·핵심 발견·제언** 블록을 식별했다.
2. [ ] **히어로**에 카테고리 배지 + 제목 + 한 줄 요약을 썼다.
3. [ ] **개요**에 도입 단락 + 2~4개 요약 카드를 채웠다.
4. [ ] **방법론**을 한 블록(또는 소수)으로 압축했다.
5. [ ] **주요 발견사항**을 번호·제목 블록으로 나누고, 각 블록에 본문 + VOC를 넣었다.
6. [ ] **제언**을 수혜자 기준 제목 + 구체적 제안 + 한 줄 요약 + 출처 문구로 마무리했다.
7. [ ] 전체 분량이 **원본 보고서보다 짧고**, 스캔 가능한지 확인했다.

## 6. HTML 변환 시 매핑 (참고)

- 이 규칙으로 정리한 **.md**는 이후 HTML 템플릿에 넣을 때:
  - `## 개요` → `<section id="overview">`
  - `## 연구 방법론` → `<section id="methodology">`
  - `## 주요 발견사항` → `<section id="findings">`
  - `## 제언 및 시사점` → `<section id="impact">`
  - 인용은 `<div class="bg-primary ...">` 퀵 인용 블록 또는 본문 내 `<p class="... italic">`
  - 발견 블록은 `border-l-4 border-primary` 카드 형태로 매핑

위 매핑은 자동화(스크립트) 또는 수동 편집 시 참고용이다.
